QUIZGAMBIT APPLICATION ANALYSIS REPORT
Generated: 2026-01-07

1. EXECUTIVE OVERVIEW
---------------------
QuizGambit is a real-time multiplayer trivia application with two distinct game modes: "Standard" (Host/Buzzer style) and "Arena" (PvP draft & tactical style). It uses a "Host-Client" architecture where one device acts as the console (Host) and others as controllers/displays (Players).

Tech Stack:
- Frontend: React (Vite), Tailwind CSS
- Backend: Supabase (PostgreSQL, Realtime, Auth, RPCs)
- State: React State + Realtime Subscriptions (No global store like Redux)
- AI: OpenAI, Gemini, Groq integration for question generation

3. GAME MODES & LOGIC
---------------------

A. STANDARD MODE (The "Jeopardy" Style)
   - Target: TV/Projector + Phones.
   - Host View (`/host` -> `HostDashboard.tsx`):
     - Lobby Creation: auto-generates 4-letter code.
     - Category Selection: Host picks categories manually OR monitors a "Player Draft".
     - Spaced Repetition: Logic ensures questions aren't repeated for the same group/host context.
   - Game Board (`GameBoard.tsx`):
     - Grid: Classic 5x5 grid of categories/points.
     - Question Modal: Show Question -> Reveal Answer -> Grade Players (+/-).
     - Features: Timer, Buzzer Lock/Unlock, Score syncing.
   - Player View (`/play` -> `PlayerView.tsx`):
     - Mobile-first interface.
     - Giant "Buzzer" button (Synced realtime with Host lock state).
     - "Draft" interface (if Player Draft is enabled) to pick categories.

B. ARENA MODE (The "PvP" Style)
   - Target: All players on own screens.
   - Host View (`/arena` -> `ArenaLobby.tsx`):
     - Sets parameters: Rounds, Cats/Round, Timer, Mystery Mode.
     - Manages "Draft Phase" (Turn-based category picking).
     - "Command Center" style UI.
   - Player View (`/play?mode=arena` -> `ArenaBoard.tsx`):
     - FULL Game Board on player device (unlike Standard).
     - Turn-Based Picking: Active player picks category/question from grid.
     - Answering:
       - Inputs: Numeric keypad or MCQ buttons.
       - Timer: Synced countdown.
     - Results: Speed-based scoring display after every question.
     - Features: Realtime rank tracking, "My Turn" notifications.

5. DATA & BACKEND STRUCTURE
---------------------------
Tables (Inferred):
- `lobbies`: Stores game state (`status`, `settings`, `arena_state`, `current_question`).
- `players`: Connected users per lobby (`score`, `name`, `metadata`).
- `questions`: Active game questions (copied from Library during draft).
- `categories_library`: Permanent storage of AI-generated content.
- `arena_answers`: Log of answers for PvP scoring logic.

Key Logic:
- Realtime Sync: Heavy use of `supabase.channel` to sync game state (phase, active question, buzzers) across clients.
- Polling Fallbacks: Implemented to handle network instability.
- RPCs: Used for complex state transitions (`next_arena_turn`, `submit_arena_answer`, `start_arena_session`).
